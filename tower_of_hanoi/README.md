# Tower of Hanoi

1, 2, 3 번의 기둥이 있다.
이 하노이의 탑 문제는 1번 기둥에서 3번 기둥으로 옮기는 문제이다.
이 과정을 텍스트가 아니 이미지로 알고 싶어서 만들었다.
아래는 원반의 개수가 2인 경우의 전체와 5인 경우의 일부이다.

형식은 ''' [원반의 번호] : 이동 전 기둥의 번호 -> 이동 후 기둥의 번호 ''' 를 따른다.
가장 길이가 짧은 원반의 번호를 1로 지정하였다.
